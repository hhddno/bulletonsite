Autoriser l'aperçu iframe sur bulletonsite.com
==============================================

Le portfolio affiche vos sites dans des mini-navigateurs (carrousel Réalisations).
Par défaut, beaucoup de sites refusent d'être encadrés → capture de secours à la place.

QUI FAIT QUOI
-------------
• Sur CHAQUE site client (Ela, Quai, ETCBC, SQCDP…) :
  ajouter l'en-tête CSP frame-ancestors qui autorise bulletonsite.

• Sur le portfolio (BulleTonSite) :
  rien de bloquant côté sortie, les sites clients doivent autoriser l'embed.

• bulletonsite.com :
  même config Vercel que bulletonsite.vercel.app (domaine custom dans Vercel → Settings → Domains).

EN-TÊTE À DÉPLOYER SUR LES SITES CLIENTS
----------------------------------------
frame-ancestors 'self' https://bulletonsite.vercel.app https://bulletonsite.com https://www.bulletonsite.com http://localhost:3000

Vercel : copier embed-templates/vercel.json à la racine du projet client.
Netlify : copier embed-templates/netlify-headers vers public/_headers

Si le site a déjà une CSP, fusionner frame-ancestors (ne pas écraser toute la politique).

FICHIERS PAR PROJET
-------------------
  maison-ela      → lamaisondela.com
  quai-des-reves  → quai-des-reves.vercel.app
  etcbc           → etcbc-charpente.com
  sqcdp           → sqcdp.vercel.app

Prompts prêts à coller : voir PROMPTS.md dans ce dossier.

Après déploiement du client : vider le cache navigateur ou attendre 1–2 min, puis recharger bulletonsite.com#projets.
